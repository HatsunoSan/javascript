O que o Javascript faz - Javascript faz praticamente TUDO!


CLIENT X SERVIDOR: Computadores e Celulares são os "Clientes".
Quando utilizo um dispositivo para entrar na internet através de um navegador, o dispositivo acessa o site percorrendo o caminho do servidor. As informações vão (dispositivo cliente pedindo autorização e informação) e volta (servidor garantindo autorização e enviando informação).

Eu entro no site. O navegador envia uma solicitação pro servidor (informação vai), solicitando arquivos do site, como vídeos, fotos, textos... E o servidor devolve uma cópia (informação volta).
O navegador então trabalha pra gerar o efeito visual, o site.

Dito isso, existem duas tecnologias nessa explicação. A tecnologia para clientes e a tecnologia para servidores, e o Javascript surgiu como uma tecnologia PARA CLIENTES! E hoje em dia, funciona até em servidores.
Mas, de forma básica, o Javascript funciona bem mais Clientside.

Vamos usar um Website whatever de exemplo.
Esse website, na grande maioria das vezes, é composto por 3 tecnologias clientside. No Serverside, também tem muitas tecnologias, mas o foco aqui é clientside.

Imaginemos um Jornal.
Um jornal precisa de conteúdo, certo? Texto, imagem, uma boa organização e um meio dele chegar ao leitor, além de estar apresentável.

Imaginemos agora 3 pessoas que trabalham nesse jornal, e que sem elas seria IMPOSSÍVEL do jornal acontecer:

A JORNALSITA é a que dá conta do texto, das imagens, do conteúdo desse jornal.

O DESIGNER é aquele que vai dar o ar de apresentação ao jornal. Como as coisas vão se apresentar ao leitor, como aquela imagem vai se enquadrar, a logo do jornal, e etc.

E O PROGRAMADOR (que nao exatamente faz sentido existir num Jornal, e entregador seria estranho, então... Programador.)
Que é quem vai configurar a rota de entrega, lidar com a segurança da entrega, e fazer todo o resto ficar funcional.

Esses 3 podem ser chamados também de:

HTML - Jornalista - Conteúdo
CSS - Designer - Design
Javascript - eNtRegaDorR/Programador

Mas importante: HTML e CSS NÃO são linguagens de programação. São tecnologias de construção de site.
